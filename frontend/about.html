About the System & Architecture

Project Overview

The Medical Research Assistant is a portfolio-grade, production-ready Full Stack AI Application designed to index and query medical literature. The application utilizes a hybrid retrieval framework combining vector similarity and keyword search, grounded checks to prevent hallucinations, and structured outputs for research summarization, clinical entity parsing, and quality evaluations.

By hosting both search indexing (FAISS and local BM25) and inference generation (Ollama Qwen3) 100% locally, this pipeline guarantees complete data privacy—a crucial compliance requirement for medical diagnostics and clinical research.

RAG Pipeline Flow

1
Document Loading & Chunking:

PDF documents are parsed, enriched with file metadata, and split into 600-character blocks with 150-character overlap.

2
Hybrid Index Retrieval:

Input queries retrieve matching segments concurrently from FAISS (vector similarity) and BM25 (keyword frequency).

3
Reciprocal Rank Fusion (RRF):

Retrieval segments are consolidated and re-ranked using an RRF algorithm (constant c=60).

4
Cross-Encoder Re-Ranking:

The top consolidated segments are scored against the query using a Cross-Encoder transformer model to prioritize core evidence.

5
Grounded Answer Synthesis:

Qwen3 synthesizes a structured answer, citing specific source PDFs and page numbers. Fallback triggers if confidence is below 0.60.

Core Technology Stack

Frontend UI HTML5, Vanilla CSS, JS Canvas
Backend Web Server FastAPI, Python
Semantic Vector Index FAISS DB, LangChain
Embedding Model BAAI/bge-large-en-v1.5
Re-ranker Model ms-marco-MiniLM-L-6-v2
Language Model Qwen3 via Ollama
